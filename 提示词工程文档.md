# 三一工业智能服务平台 — 提示词工程文档

| 原则 | 说明 |
|------|------|
| **先读后写** | 每次生成代码前先读取目标文件上下文 |
| **批量并行** | 独立文件同时创建/修改，减少交互轮次 |
| **验证闭环** | 每次改动后执行 `npm run build` 确保编译通过 |
| **一致性优先** | 代码风格、命名、组件模式始终与现有代码对齐 |
| **最小改动** | 优先用 `edit` 而非 `write`，只改需要改的部分 |

### 1.3 项目技术栈识别指令

在向 AI 描述项目时，应提供以下技术栈签名：

```
Vue 2.7 + Element UI 2.x + Vue Router 3.x + Axios
Element UI 组件库，使用 slot-scope 模板语法
element-ui/lib/theme-chalk/index.css 主题
```

## 2. 提示词框架

### 2.1 通用指令前缀

在每个任务前添加以下指令约束 AI 行为：

```
你是一个资深 Vue 2 + Element UI 前端工程师。
```

## 3. 各阶段提示词模板

### 3.1 项目探索阶段

```
看一下我的这个项目
```

> 目的：让 AI 自动读取目录结构、分析技术栈、理解当前状态。  
> 输出：目录树 + 技术栈识别 + 现有功能评估。  
> 后续指令根据 AI 输出的内容决定下一步方向。

### 3.2 新增功能页面

模板：

```
在 src/views/ 下新增 XX.vue 页面，功能要求：
1. 数据表格展示，支持分页（参考 WorkOrderList.vue 的分页逻辑）
2. 顶部筛选栏，包含 XX/XX/XX 筛选条件
3. 新增/编辑弹窗（el-dialog），表单包含 XX/XX/XX 字段
4. 删除确认（$confirm）
5. API 调用统一走 src/api/XX.js（需要新增该文件）
6. 样式复用 global.css 中的 .page-container / .filter-bar / .table-container / .pagination-wrapper
7. 不要添加注释
```

同时创建 API 文件：

```
在 src/api/ 下新增 XX.js，包含以下 API 封装：
- getXXPage(params)    GET /api/v1/XX/page
- getXXDetail(id)      GET /api/v1/XX/{id}
- createXX(data)      POST /api/v1/XX
- updateXX(data)      PUT /api/v1/XX
- deleteXX(id)        DELETE /api/v1/XX/{id}
使用 import request from '../utils/request'
```

### 3.3 注册路由 + 菜单

模板：

```
1. 在 src/router/index.js 中添加：
   imports + { path: 'XX', name: 'XX', component: XX, meta: { title: 'XX' } }

2. 在 src/layout/Layout.vue 中添加 el-menu-item 和 breadcrumbName 映射
```

### 3.4 添加 Mock 数据

模板：

```
在 src/mock.js 中为 XX 模块添加 Mock 数据：
1. 在 _ 对象中添加 XX 数组，包含 5-8 条示例数据
2. 在 mock() 函数中添加 URL 匹配和数据处理逻辑
3. 支持分页（page函数）和筛选（params 参数过滤）
4. 支持 CRUD（增删改查操作对应的内存数组更新）
```

### 3.5 样式优化

模板：

```
对 global.css 进行样式优化，要求：
1. 建立完整的 CSS 变量系统（颜色/阴影/圆角/间距/过渡）
2. 覆盖 Element UI 默认样式（表格/卡片/对话框/表单/标签/分页等）
3. 添加全局布局类（page-container / filter-bar / table-container 等）
4. 设计响应式断点（1200px / 768px）
5. 统计卡片/通知列表/空状态等通用组件样式
6. 不要添加注释

然后对整个项目的所有 view 页面进行以下处理：
1. 将 inline style 抽取为 scoped CSS 类
2. 改用 global.css 中的 CSS 变量（var(--color-xxx)）
3. 统一使用 .table-container + .table-toolbar 结构
4. 用 .stat-grid / .dashboard-row / .dashboard-col-* 替代 el-row/el-col
```

### 3.6 添加初始数据（Mock）

模板：

```
给项目加一些初始数据，不然老是报 404

要求：
1. 创建 src/mock.js，使用 axios adapter 拦截所有 API 请求
2. 覆盖全部现有 API 端点，返回模拟数据
3. 数据量充足（5-10 条/模块），字段完整
4. 支持筛选、分页、CRUD 操作
5. 模拟 80-200ms 网络延迟
6. 在 src/main.js 中通过 import installMock from './mock' 启用
```

### 3.7 输出文档

模板：

```
输出一份部署文档 / 提示词工程文档

要求：
1. 结构清晰，分章节带目录
2. 包含实际可执行的命令（npm run dev / mvn package 等）
3. 基于项目实际目录结构和配置编写
4. 场景覆盖全面（开发/测试/生产）
```

## 4. 上下文管理策略

### 4.1 多文件读取

当需要理解多个文件后做改动时，一次性发送所有读文件请求：

```
读取以下文件：
- src/router/index.js
- src/layout/Layout.vue

```

> AI 支持并发读文件，读完后再统一分析，减少不必要的轮次。

### 4.2 批量创建

当需要创建多个独立的新文件时，一次性发送所有写文件请求：

```
同时创建：
1. src/views/XX.vue（页面组件）
2. src/api/XX.js（API 封装）
3. src/mock.js 追加 XX 模块数据
```

### 4.3 渐进式修改

不要一次性提所有需求，按依赖顺序排列：

```
第 1 轮：创建页面组件 + API 文件
第 2 轮：注册路由 + 菜单
第 3 轮：添加 Mock 数据
第 4 轮：npm run build 验证
```

### 4.4 使用 TodoList 管理多步骤

当任务超过 3 个步骤时，让 AI 使用 TodoWrite 工具创建任务清单：

```
把以上需求列一个 todo 清单，逐步完成
```

## 5. 代码生成约束规则

### 5.1 Vue 2 模板语法

```html
<!-- 正确：slot-scope -->
<template slot-scope="{ row }">
  <span>{{ row.name }}</span>
</template>

<!-- 错误：v-slot（Vue 3 语法） -->
<template v-slot="{ row }">
```

### 5.2 Element UI 版本约束

```javascript
// 正确：Vue 2 + Element UI 2.x
import ElementUI from 'element-ui'
import 'element-ui/lib/theme-chalk/index.css'
Vue.use(ElementUI)

// 错误：Element Plus（Vue 3 版本）
```

### 5.3 API 调用模式

```javascript
// 使用封装的 request，非直接 axios
import request from '../utils/request'

export function getData(params) {
  return request({ url: '/api/v1/xxx', method: 'get', params })
}

// 响应格式：res.code / res.data
const res = await getData(params)
if (res.code === 200) {
  this.list = res.data.list || []
  this.total = res.data.total
}
```

### 5.4 Mock 数据格式

```javascript
// 响应结构
{ data: { code: 200, message: 'success', data: actualData } }

// 分页结构
{ list: [...], total: N, pageNum: 1, pageSize: 10 }

// adapter 返回格式（重要）
{ data: {...}, status: 200, statusText: 'OK', headers: {...}, config: {...} }
```

### 5.5 样式引用策略

```css
/* 全局样式在 App.vue 中引入 */
@import './assets/global.css';

/* 页面级样式使用 scoped + CSS 变量 */
<style scoped>
.my-class { color: var(--color-text); }
</style>

/* 禁止内联 style（少量动态值除外） */
```

### 5.6 构建验证

```bash
npm run build
```

> 每次修改后必须执行，确保无编译错误。仅警告（WARNING）可接受。

## 第1章 需求分析

### 1.1 项目背景

面向三一集团（SANY）的工业智能服务平台，旨在通过数字化手段整合设备运维、报修工单管理和客户服务三大核心业务，实现设备全生命周期管理，提升售后服务效率与客户满意度。

### 1.2 核心用户角色与功能

| 角色 | 定义 | 核心功能 |
|------|------|----------|
| **客户** | 设备购买方/使用方，即三一的终端客户 | 设备档案查看、在线报修、报修进度跟踪、保养提醒接收、服务评价、备件申购 |
| **运维人员** | 一线服务工程师/维修技师 | 工单接单/抢单、现场维修、备件领用、服务记录填写、完工反馈、移动端签到 |
| **管理员** | 三一内部服务调度/管理人员 | 工单派发/调度、人员排班、备件库存管理、服务记录审核、数据统计看板、系统配置 |

### 1.3 核心业务流程

#### 1.3.1 设备报修流程

```
客户提交报修 → 系统自动生成工单 → 管理员审核派单 → 运维人员接单
    → 现场维修处理 → 客户验收确认 → 服务记录归档 → 客户评价
```

#### 1.3.2 工单处理流程

```
工单创建（客户/管理员） → 工单调度（智能派单/手动派单）
    → 运维人员接单（确认/转单） → 出发签到 → 到达签到
    → 故障诊断 → 维修/更换处理 → 完工确认 → 客户签字验收
    → 服务回访 → 工单关闭
```

#### 1.3.3 保养提醒流程

```
设备档案登记 → 设置保养周期（按工时/按日历）
    → 到期前自动推送提醒 → 客户确认 → 生成保养工单
    → 派单处理 → 保养执行 → 保养记录归档
```

#### 1.3.4 服务记录流程

```
工单执行过程中自动采集：故障现象、处理措施、更换备件、工时
    → 管理员审核 → 知识库沉淀（故障库+解决方案库）
    → 统计分析（故障率、响应时长、一次修复率）
```

### 3.3 核心设计原则

1. **前后端分离**：后端只提供 RESTful API，前端独立部署互不依赖
2. **RESTful 规范化**：统一 API 命名 /api/v1/xxx，统一返回结构 R< T >
3. **统一异常处理**：全局 `@RestControllerAdvice` 捕获异常，返回标准错误码
4. **JWT Token 鉴权**：无状态认证，Token 携带角色权限信息
5. **数据权限隔离**：客户只能看自己的设备；运维人员看分配工单；管理员全局
6. **接口幂等性**：关键接口（创建工单、扣减库存）通过唯一请求号防重复

## 第4章 核心数据库表结构设计

### 4.1 总体设计约定

- **数据库引擎**：InnoDB
- **字符集**：`utf8mb4`，排序规则 `utf8mb4_unicode_ci`
- **主键策略**：自增 BIGINT / Snowflake ID（分布式场景）
- **软删除**：所有表增加 `deleted` 字段（TINYINT，0未删/1已删）
- **时间字段**：`create_time`、`update_time` 由 MyBatis-Plus 自动填充
- **命名规范**：表名 `tbl_模块_含义`，字段 `snake_case`

### 4.2 用户表（tbl_sys_user）

```sql
CREATE TABLE tbl_sys_user (
    id              BIGINT          NOT NULL AUTO_INCREMENT  COMMENT '主键ID',
    username        VARCHAR(64)     NOT NULL                 COMMENT '登录账号',
    password        VARCHAR(256)    NOT NULL                 COMMENT '加密密码',
    real_name       VARCHAR(64)                              COMMENT '真实姓名',
    phone           VARCHAR(20)                              COMMENT '手机号',
    email           VARCHAR(128)                             COMMENT '邮箱',
    avatar          VARCHAR(512)                             COMMENT '头像URL',
    user_type       TINYINT         NOT NULL DEFAULT 0       COMMENT '用户类型 0-管理员 1-运维人员 2-客户',
    status          TINYINT         NOT NULL DEFAULT 1       COMMENT '状态 0-禁用 1-启用',
    customer_id     BIGINT                                   COMMENT '所属客户ID（客户角色时关联）',
    deleted         TINYINT         NOT NULL DEFAULT 0       COMMENT '逻辑删除标记',
    create_time     DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time     DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (id),
    UNIQUE KEY uk_username (username),
    KEY idx_phone (phone),
    KEY idx_customer_id (customer_id),
    KEY idx_user_type (user_type)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='系统用户表';
```

## 第8章 安全设计

1. **认证方式**：JWT（access_token 2h过期 + refresh_token 7d）
2. **密码存储**：BCrypt加密，不存明文
3. **接口防篡改**：敏感接口校验签名
4. **防重复提交**：Redis分布式锁 + 唯一请求号
5. **SQL注入防护**：MyBatis参数绑定，禁止拼接SQL
6. **XSS防护**：前端输入过滤，后端输出编码
7. **数据脱敏**：手机号、身份证等敏感字段查询时脱敏
8. **操作审计**：所有增删改操作记录日志表
9. **HTTPS**：生产环境强制HTTPS

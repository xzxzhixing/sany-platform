3# 三一工业智能服务平台 — 项目总结

> 版本：v2.0.0 | 日期：2026-05-26

---

## 1. 项目概述

三一工业智能服务平台是一个面向工程机械后市场的设备运维与客户服务管理系统，涵盖工单管理、设备管理、客户管理、备件管理、服务报告、数据统计等核心业务模块。

### 项目愿景

通过数字化手段整合设备运维、报修工单管理和客户服务三大核心业务，实现设备全生命周期管理，提升售后服务效率与客户满意度。

## 5. 关键设计

### 5.1 样式系统

基于 CSS 变量建立完整设计令牌体系：

```css
:root {
  --color-primary: #1890ff;
  --color-bg: #f0f2f5;
  --color-text: #1d1e23;
  --shadow-sm: 0 1px 2px rgba(0,0,0,.06);
  --radius-md: 8px;
  --sidebar-width: 220px;
  --topbar-height: 56px;
  --transition-normal: 0.3s;
  /* 共 20+ 个变量 */
}
```

覆盖 Element UI 全部常用组件（表格/卡片/对话框/表单/按钮/标签/分页/进度条），实现一致的视觉风格。

### 5.2 布局系统

- 固定侧边栏（220px，可折叠至 64px），深色主题，菜单项用蓝色渐变高亮
- 顶部栏粘性定位，含面包屑、通知铃铛、用户头像
- 内容区域自适应，页面统一 `page-container` 容器

### 5.3 Mock 数据层

采用 axios adapter 方案，在 HTTP 请求的最底层拦截：

```javascript
axiosInstance.defaults.adapter = function (config) {
  const resp = mock(config)   // 匹配 URL 返回模拟数据
  if (resp) return new Promise(resolve => setTimeout(() => resolve(resp), delay))
  return origAdapter(config)  // 未匹配则转发真实后端
}
```

特点：
- **无侵入**：不修改任何业务代码
- **可开关**：注释 `main.js` 中一行即可关闭
- **低延迟**：80-230ms 随机延迟，模拟真实网络
- **全功能**：支持筛选、分页、CRUD 全部操作

### 5.4 路由与导航

11 个路由统一注册在 `Layout` 子路由下，侧边栏菜单、面包屑映射同步维护。`el-menu` 配合 `router` 属性实现自动高亮。

## 8. 启动方式

### Mock 模式（无需后端）

```bash
cd frontend
npm install
npm run dev
# 访问 http://localhost:9528
```

### 完整模式（前后端+数据库）

```bash
# 1. 初始化数据库
mysql -u root -p < sql/work_order.sql

# 2. 启动后端
cd backend
mvn clean package -DskipTests
java -jar target/sany-platform.jar

# 3. 启动前端（注释 main.js 中的 Mock 启用）
cd frontend
npm run dev
```
